# Stage 1 Discovery: Pagination

## 1. Arab48 pagination

`crime_pipeline/scrapers/arab48.py` currently builds only:

```py
https://www.arab48.com/بحث?searchText=<quote_plus(query)>
```

Live probe with `httpx`, Mozilla UA, `trust_env=False` because the sandbox proxy points at `127.0.0.1:9`:

- `https://www.arab48.com/بحث?searchText=%D9%82%D8%AA%D9%84` -> 200, 20 unique article links, pager links `1 2 3 4 5 ... 722`.
- `https://www.arab48.com/بحث?page=2&searchText=%D9%82%D8%AA%D9%84` -> 200, 20 unique article links, next link to `page=3`.
- `https://www.arab48.com/بحث?page=3&searchText=%D9%82%D8%AA%D9%84` -> 200, 20 unique article links.

The hrefs emitted by the page use pattern:

```text
page 1: /بحث?searchText=<query>
page N: /بحث?searchText=<query>&page=N
```

Query parameter order is accepted either way. Normal search result pages contain 20 result articles. The pager exposes a max-page indicator as the last numeric page link; for query `قتل` on 2026-05-11 it was `722`. Out-of-range pages can return a generic page with only a few latest-site article links and no pager, so do not trust HTTP 200 alone as success. Stop on "no new URLs".

Recommended shape:

```py
async def discover(self, query, date_from, date_to,
                   max_results=50, max_pages=5):
    from_dt = datetime.fromisoformat(date_from).replace(tzinfo=timezone.utc)
    to_dt = datetime.fromisoformat(date_to).replace(tzinfo=timezone.utc)
    discovered, seen = [], set()

    async with _make_client() as client:
        for page in range(1, max_pages + 1):
            if len(discovered) >= max_results:
                break
            search_url = (
                f"{_SEARCH_URL}?searchText={quote_plus(query)}"
                if page == 1
                else f"{_SEARCH_URL}?searchText={quote_plus(query)}&page={page}"
            )
            try:
                resp = await self._get(client, search_url)
                resp.raise_for_status()
            except (_RetryableHTTPError, httpx.HTTPError) as exc:
                logger.error("Arab48 discover page %d: %s", page, exc)
                break

            soup = BeautifulSoup(resp.text, "lxml")
            page_new = 0
            for a_tag in soup.find_all("a", href=True):
                href = a_tag["href"]
                if not any(m in href for m in _ARTICLE_LINK_MARKERS):
                    continue
                full_url = urljoin(_BASE_URL, href)
                parsed = urlparse(full_url)
                if parsed.netloc and parsed.netloc != self.base_domain:
                    continue
                if full_url in seen or _is_non_homicide_path(parsed.path):
                    continue
                seen.add(full_url)
                # reuse existing title/pub_date extraction and date filter
                # append DiscoveredUrl exactly as current code does
                page_new += 1
                if len(discovered) >= max_results:
                    break
            if page_new == 0:
                break
```

Risks: search sorting appears date-desc by default, but advanced filters can affect ordering; keep the current date filter per item. `_RetryableHTTPError` already retries 429/5xx with exponential backoff, and `_sleep()` runs before each page.

## 2. Ynet rate limit

`ynet.py` already avoids Google News RSS result pagination by splitting the date range into 48h windows and bisecting saturated windows (`len(raw_items) >= 90`) down to 3h / depth 4. Google News RSS itself does not expose page cursors here; additional pagination is not the missing piece.

Bisection should catch most articles as long as each RSS response is complete below the saturation threshold. Failure modes:

- A true result set above Google RSS's hidden cap but below/around the chosen `90` threshold can still be truncated.
- A 429 raises `_RateLimitStop`, aborts remaining windows, and returns partial results. This is explicit but loses later windows.
- Non-429 transient errors return `[]`, logging "possible silent empty"; those windows are not retried or bisected.

Recommendation: keep bisection, but replace hard stop on 429 with retry/backoff plus checkpointable continuation, or return discovery metadata marking partial results. Also retry empty/non-200 windows once or twice before accepting empty.

## 3. Cross-scraper interface

Common contract:

```py
async def discover(self, query: str, date_from: str, date_to: str,
                   max_results: int = 50, max_pages: int | None = None
) -> list[DiscoveredUrl]:
```

`max_results` is always the final returned unique URL cap. `max_pages` is a source-specific discovery effort cap: Arab48 search pages; Ynet Google News windows fetched, or `None` for current full-range behavior. Pipeline/settings should pass the same optional `max_pages` to all scrapers, defaulting Arab48 to `5` and Ynet to `None` or a high operational cap.
