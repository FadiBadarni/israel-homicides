## HIGH

**`rejected_unrelated_articles` silently dropped by `CanonicalCaseSchema(**d)` rehydration**

- `crime_pipeline/enrichment/quality_pass.py:680` — `run_quality_pass` writes `case["rejected_unrelated_articles"]` to the dict.
- `crime_pipeline/pipeline.py:649` — `_run_cleanup` terminates with `[CanonicalCaseSchema(**d) for d in case_dicts]`. Pydantic's default `extra="ignore"` silently drops any key not declared in the model, so `rejected_unrelated_articles` disappears.
- `crime_pipeline/models.py` — the field was NOT added to `CanonicalCaseSchema` alongside the six other new fields.
- **Impact**: `reevaluate_rejected_articles` (the last step in `run_quality_pass`) sets the field, but since cleanup runs only once per pipeline execution, the field is only read within the same pass and never surfaces in the exported JSON. Operators cannot audit which articles were rejected. The correctness of merging is unaffected; this is an audit-trail regression.
- **Fix**: Add `rejected_unrelated_articles: list[dict[str, Any]] = Field(default_factory=list, ...)` to `CanonicalCaseSchema` alongside `dropped_invalid_sources`.

**Transitive provenance emits `jaro_score: 0.0` — misleads operators**

- `crime_pipeline/enrichment/reconciler.py:279-283` — When a 3-way cluster is formed where A↔B and B↔C were directly compared but A is chosen as canonical (most sources), the provenance entry for C records `reason: "transitive", jaro_score: 0.0`. The 0.0 is wrong: C was actually matched at the B↔C jaro score (≥0.85). The transitive fallback IS reachable: verified with `Yosef Ben David` / `Yossi Hanun` / `Yosef Hanun` at threshold 0.85 (pairs (0,1) and (1,2) recorded; (0,2) not, yet C ends up in A's cluster).
- **Impact**: The audit JSONL for the reconcile stage shows `jaro_score: 0.0` for transitively-merged cases, which could cause an operator to distrust the merge or incorrectly assess merge confidence.
- **Fix**: Instead of 0.0, propagate the best-known jaro from any pair that links `weak_idx` into the cluster. One approach: build a `case_jaro: dict[int, float]` that maps each non-root member to its highest directly-observed jaro score, populated during the union loop, then use that in `_provenance_for`.
