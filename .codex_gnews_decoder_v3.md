Use `googlenewsdecoder` to replace the broken DOM-scraping redirect resolver; it already tracks Google's batchexecute flow and supports `/rss/articles` plus `/articles` fallbacks.

Recommended library: [`googlenewsdecoder`](https://pypi.org/project/googlenewsdecoder/) `0.1.7`, released Jan 18, 2025. It is the maintained, purpose-built package among the named candidates I could verify, has PyPI/GitHub source links, Python >=3.9 support, proxy/rate-limit handling notes in 0.1.7, and PePy reports ~1.16M total downloads; PyPI itself does not expose monthly downloads on the project page.

Replacement code:

```python
# add near imports
from googlenewsdecoder import GoogleDecoderAsync

_GNEWS_DECODE_CACHE: dict[str, str] = {}
_GNEWS_DECODE_LOCK = asyncio.Lock()
_GNEWS_DECODE_INTERVAL_SECONDS = 1

async def _resolve_google_url(
    client: httpx.AsyncClient, google_url: str
) -> str | None:
    """Resolve a Google News URL to a canonical ynet.co.il URL."""
    del client  # kept for the existing call signature

    parsed = urlparse(google_url)
    parts = parsed.path.strip("/").split("/")
    if parsed.netloc != "news.google.com" or not {"articles", "read"} & set(parts):
        return None

    cached = _GNEWS_DECODE_CACHE.get(google_url)
    if cached is not None:
        return cached

    async with _GNEWS_DECODE_LOCK:
        cached = _GNEWS_DECODE_CACHE.get(google_url)
        if cached is not None:
            return cached

        try:
            async with GoogleDecoderAsync() as decoder:
                result = await decoder.decode_google_news_url(
                    google_url,
                    interval=_GNEWS_DECODE_INTERVAL_SECONDS,
                )
        except Exception as exc:
            logger.warning("gnews redirect: decoder raised for %s - %s", google_url, exc)
            return None

    if not result.get("status"):
        logger.warning("gnews redirect: decode failed for %s - %s",
                       google_url, result.get("message"))
        return None

    decoded_url = result.get("decoded_url")
    if not isinstance(decoded_url, str):
        logger.warning("gnews redirect: missing decoded_url for %s", google_url)
        return None

    if not decoded_url.startswith(("https://www.ynet.co.il/", "https://ynet.co.il/")):
        logger.warning("gnews redirect: unexpected domain in %s", decoded_url)
        return None

    _GNEWS_DECODE_CACHE[google_url] = decoded_url
    return decoded_url
```

Dependency line in `pyproject.toml`:

```toml
"googlenewsdecoder>=0.1.7",
```

Test scenarios: valid `news.google.com/rss/articles/...` for Ynet returns the decoded `https://www.ynet.co.il/...`; valid `news.google.com/read/...` also resolves; non-Google or malformed URLs return `None`; decoder `{status: False, message: ...}` or exception returns `None` and preserves `google_redirect_unresolved` caller behavior; decoded non-Ynet URL returns `None`.

Risks: the async decoder still depends on Google's private batchexecute contract, so keep failures soft and do not keep the old broken BeautifulSoup fallback because it only adds latency. The lock plus 1s interval prevents accidental parallel decode bursts; the in-memory success cache avoids duplicate work during one run without schema churn.
