## 3. Verify CLI Architecture

The `verify` command will provide the quantitative backbone for evaluating pipeline performance.

### CLI Interface
```bash
python -m crime_pipeline verify --truth data/truth_2025.jsonl --run output/run_X.json
```

### Truth File Format (JSON Lines)
JSONL is preferred over CSV to handle multi-script names (Hebrew/Arabic) and nested metadata without escaping issues.
```json
{"city": "Nazareth", "victim_name_he": "...", "victim_name_ar": "...", "incident_date": "2025-05-11"}
```

### Matching Algorithm
Reuse the existing `is_same_incident()` logic from `crime_pipeline/enrichment/enricher.py`. This ensures consistency between how the pipeline merges cases and how it evaluates them.
- **Identity Gate:** Jaro-Winkler (threshold 0.70) on names + City Match (via Gazetteer) + Date Window (±5 days).

### Evaluation Metrics
- **Recall:** % of Abraham Initiatives victims found by the pipeline.
- **Precision:** % of pipeline-detected cases that are "True" homicides in Arab society.
- **FN List:** Missed cases (Crucial for identifying gaps in keyword recall).
- **FP List:** Noise (e.g., Jewish-sector homicides or non-fatal shootings accidentally included).
